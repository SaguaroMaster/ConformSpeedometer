Usage

Date Selection

Using this function the user can select a day from the calendar for which the data shown will be calculated from.
A "day" starts from 06:00 on the selected day and last until 06:00 the next day.
This is done so that the displayed graphs align with the starts/ends of the shifts.

Current Speed (Numeric value)

Current speed of the line on which the device is installed.
This value represents the last 5 minutes average.
Updates every 5 minutes, as every other measurement.
It is calculated from the frequency of impulses that are generated by metal pieces mounted on a wheel that is rotated by the passing wire.
These metal pieces pass in front of an inductive sensor that then generates electric impulses.

Current Length

The maximum length the device logs from every 5 minute interval.
It is reset when the operator resets the counter on the screen of the device.
Reaction of the web interface can be delayed by up to 5 minutes due to the update interval.

Last Reading

The last time the readings have been written into the database.
This occurs every 5 minutes.
The data seen on the web interface is calculated based on the values in the database, not directly from the sensors (as it is on the screen of the device).

Average Speed

This value represents the average speed of the line for the selected day, including stops.

Downtime 24h/30d

This value is the sum of the duration of all stops that are above a certain time limit (currently 10s).
There are 2 values, one that represents downtime for the selected day, on that represents the past 30 days starting from the selected day.
A limitation of this function is that the 24 hour version does not include downtime where the line was stopped before a shift change (e.g. 5:45) and was started after it (e.g. 6:30).
Format is HOURS:MINUTES:SECONDS.

Times Stopped 24h/30d

The given line was stopped this many times in the past 24hours or 30 days.
This only includes stops that are above a certain time limit (currently 10s).
By clicking on this value a list is shown detailing all the downtimes down to second accuracy (also includes values below the 10 second limit).
A limitation of this function is that the 24 hour version does not include downtime where the line was stopped before a shift change (e.g. 5:45) and was started after it (e.g. 6:30).
If the user wishes to also see these stops, a look at the 30d data will be neccessary.

Line Speed (Chart)

This chart represents the line speed for the given line.
Every data point is the average of the past 5 minutes data.
Given the averaging, downtime that last less than 10 minutes is not guaranteed to be shown on this chart.

Meters Made / Month

This chart serves information about how many meters of wire were made on the given line in the past month, in 1 month intervals, for the past year.

Length logged

Every data point on this chart represents the maximum length that was measured in the past 5 minutes.
This is done so that the exact length at which the operator reset the counter could be determined if neccessary.
Using this graph it is possible to see how many spools were manufactured on the given day (Conform only).

Alarm Setting

Here the user can see to what setting the operator has set the alarm limit.
Above this limit the alarm will sound.

Download database

Using this menu point the user can download the database in .db fomrat (library used is sqlite3) for backing it up or extracting data from it for custom evaluation.
